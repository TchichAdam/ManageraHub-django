\section*{Annexe A : Guide de déploiement sur Oracle Cloud}

Pour déployer ManageraHub en production sur une machine virtuelle (VM) Oracle Cloud, veuillez suivre ces étapes principales :

\subsection*{1. Configuration de l'instance Oracle Cloud}
\begin{itemize}
    \item Création d'une instance Ubuntu (Always Free Eligible) via la console OCI.
    \item Ouverture des ports 80 (HTTP) et 443 (HTTPS) dans la Virtual Cloud Network (VCN).
    \item Connexion sécurisée à la VM via une paire de clés SSH privées.
\end{itemize}

\subsection*{2. Étapes de configuration du serveur}
\begin{lstlisting}[language=bash, caption=Installation des paquets et configuration système]
# 1. Mise a jour du systeme et installation des dependances
sudo apt update && sudo apt upgrade -y
sudo apt install python3-pip python3-venv git nginx -y

# 2. Ouverture des ports du pare-feu local (Ubuntu iptables)
sudo iptables -I INPUT 6 -p tcp --dport 80 -j ACCEPT
sudo iptables -I INPUT 6 -p tcp --dport 443 -j ACCEPT
sudo netfilter-persistent save

# 3. Clonage du projet et configuration Python
git clone https://github.com/TchichAdam/managerahub-django.git
cd managerahub-django
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# 4. Preparation de la base de donnees et des fichiers statiques
python3 manage.py makemigrations
python3 manage.py migrate
python3 manage.py collectstatic --no-input
\end{lstlisting}

\subsection*{3. Lancement avec Gunicorn et Nginx}
L'application est servie en arrière-plan par \texttt{Gunicorn} (géré via \texttt{systemd}) et le trafic web est géré par \texttt{Nginx} agissant comme un reverse-proxy vers Gunicorn.
Les requêtes parviennent sur l'IP publique d'Oracle Cloud associée au domaine \url{http://51-170-143-223.sslip.io}.

\section*{Annexe C : Validations de formulaires et logique de sécurité}
Pour garantir la cohérence des données, notamment pour le marché marocain, des mécanismes de validation stricts ont été mis en œuvre.

\subsection*{1. Validateur personnalisé d'ICE marocain (forms.py)}
L'ICE (Identifiant Commun de l'Entreprise) doit être composé de 15 chiffres. Voici comment ce contrôle est géré dans notre formulaire de modification du profil de l'entreprise :

\begin{lstlisting}[language=Python, caption=Validation de l'ICE dans CompanyProfileForm]
from django import forms
from .models import CompanyProfile

class CompanyProfileForm(forms.ModelForm):
    class Meta:
        model = CompanyProfile
        fields = [
            "company_name",
            "industry",
            "company_size",
            "phone_number",
            "city",
            "country",
            "website",
            "description",
            "logo",
            "background_image",
            "ice",
            "rc_number",
            "legal_document",
        ]
        
    def clean_ice(self):
        ice = self.cleaned_data.get("ice")
        if ice:
            ice = ice.strip()
            if not ice.isdigit() or len(ice) != 15:
                raise forms.ValidationError("The ICE number must contain exactly 15 digits.")
        return ice
\end{lstlisting}
